- web基础
    + web系统的原理
        + 首先输入url，首先dns服务器根据输入的url的域名进行解析，转换成对应的ip地址，向对应的地址发送请求，对应的服务器收到请求后根据请求的内容进行相应，将对应的各种数据包相应发送给的客户端（浏览器）（主要是一些html，gzip，js等包），浏览器将收到的包进行渲染变成可视化的界面
    + url的组成：协议，ip，端口，资源路径
        + https://www.baidu.com  -- 端口为默认隐藏可以不写，资源路径已经在服务器配置文件中固定，也可以不用管，一般默认访问是 index.XX
        + http --- 80   https -- 443

- http协议
    + 默认端口 80
    + https = http + ssl（完全套接字）
    + 请求报文 -- request
        + 方法-url-协议版本
            + get(请求方式) /oa/index.html  http/1.1
        + 请求头
            + accept image/gif, */* -- 当前客户端可以处理的数据类型
            + accept-encoding gzip  -- 当前客户端可以处理压缩格式
            + accept-language zh-cn  -- 字符集支持
            + connection  keep-alive 支持链接的方式
            + host localhost 主机地址
            + 浏览器操作系统内核版本等信息
            + 空行 --分割
        + 请求正文
            + username=test1&password=123456 -- 正文
    + 响应报文 -- response
        + 响应头部
            + http/1.1 200 ok ----- 响应状态码
            + accept-ranges bytes 可接受的数据类型
            + connection keep-alive 
            + 长度
            + content-type text/html 链接内容的类型为text/html格式
            + 时间
            + keep-alive 时间
            + 服务器类型内核等信息
            + 空行 --分割
        + 响应正文
            + 如果是html就是一个html内容
    + 请求方式的区别
        + get --- 获取  http://192.168.3.117:8080/oa/login?username=test1&password=123456
        + post -- 提交 --- insert操作   http://192.168.3.117:8080/oa/login 正文在消息体里
        + head --- 获取头部信息
        + put -- 修改 -- update操作
        + delete --- 删除
        + get和post的区别？？？？
            + get是获取，post是提交
            + get类型是将正文部分拼接到url中一起传输的，post是将正文部分封装到body中的
            + get传递是有限的1kb左右，post的无限的
            + get只需要一次，post在传递需要2次
    + 响应状态码
        + 1XX -- 指示信息--表示请求已接收，继续处理
        + 2XX -- 成功
        + 3XX -- 重定向 301 302
        + 4XX -- 客户端错误  403  404
        + 5XX -- 服务端错误 500  503 
    + http和https的区别
        + https就是http安全的版
        + http要想变成https协议需要到ca申请证书进行处理
        + http默认80端口，https默认443端口
        + http是超文本传输协议，信息是明文传输，https则是具有安全性的ssl加密传输协议
        + http的连接很简单，是无状态、无连接；HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，比http协议安全
    + http1.0  1.1  2.0的区别  --- 现在还是1.1版本
        + 1.0和1.1  长连接，缓存处理，错误状态码增加
        + 1.1和2.0  头部压缩，多路复用，服务器推送

            + cookie
                + 判断访问者是否合法，是否需要再次校验，保留一个用户的合法状态，合法下一步，不合法首页 --- 请求头
                + 保存在本地浏览器中
                + 单个cookie不能超过4k，一般最多保留20个
                + 不安全容易被获取，黑客可以获取进行欺骗访问
                + 对服务压力小
                + 测试时，关注cookie信息正确性，删除后能否正常访问，刷新后是否有更新
            + session  -- 会话
                + 保存在服务器内存上
                + 关注失效时间，正确性
                + 一般像大型的像电商类一般使用cookie不适用session
                + 相对安全
            + cache -- 缓存
                + 提高响应速度，针对已经获取的信息进行重复使用
                + 在前端性能优化上
            + token -- 令牌  
                + 用户标识你身份的一串秘钥
                + 在接口测试中，前后端分离的系统很常用
