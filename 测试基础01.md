## 软件生命周期
- 市场需求调研
- 可行性研究--以收益为目的；时间，范围，成本，风险
- 产品项目立项
    + 构建产品或者研发小组
    + 指定产品运作计划
- 需求调研开发
    + 经过确认得到-----SRS需求规格说明书----FRS----PRD（一般由产品经理编写）
    + 包括需求调研，需求开发，需求评审，需求管理
- 设计开发测试
    + 设计---规划一下输出设计文档
    + 开发---编码
    + 测试---检验是否合格---和需求一致（用户满意）
    + HLD（描述一些模块的信息，接口的一些文档）和LLD（设计软件的内部逻辑，数据表结构，单元的逻辑文档）
- 发布运行维护
    + 上线----产品发布
        + 测试通过，用户验收通过，会在用户量少的时候，将测试通过的软件发布到用户环境上去
        + 用户量少------一般是0-4点
        + 休息好
        + 回滚
        + 提前准备好测试资源，以及数据备份
    + 运行---产品在用户网络正常运行
    + 维护---在运行过程中，发现问题处理，升级，打补丁
