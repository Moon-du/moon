## 研发和测试模型
- 研发模型
    + 瀑布模型
        + 计划---产品的计划，SRS
        + 需求分析---评审报告，SRS
        + 设计----HLD LLD
        + 编码----产品
        + 测试----测试计划，测试方案，测试用例，测试报告
        + 运行维护----用户手册（实施），运维手册（开发）
        + 缺点
            + 测试介入较晚，如果发现是需求/计划的问题，修复成本非常高
            + 之前的软件很小，功能不复杂，开发简单，现在的软件很复杂
            + 由于是线性的用户只能在末期才能看到结果，增加风险
            + 不利于需求的变更
            + 是只有将产品做完成后才一次性交付给用户
        + 优点
            + 适用于小项目，逻辑不复杂的项目
            + 应为每个阶段都单独进行，因此会在每个阶段分析的很透彻
    + 原型模型
        + 可以让用户在前期就能看到产品的原型
        + 给项目组开发创建一个模型
    + 螺旋模型
        + 强调风险控制
		+ 成本相对较高，只适合大型项目
    + RUP模型
        + 对大型系统降低实施的复杂度
        + 早期可以提出变更修复，控制风险
        + 不适用于功能太过紧密的系统
        + 已确定的功能不能变更
    + 敏捷测试模型
        + 快，版本迭代快
        + 将一个大项目分成很多可以独立运行的小项目，单独开发上线
        + 及时响应市场变化---可以变更
        + 迭代周期较短一般为2周左右

- 小工具---svn-版本管理工具---有git工具

    + 如何上传，更新
    + 命令更新--svn  up
