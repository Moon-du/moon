## 软件测试原则
- 证明软件存在缺陷--测试是有风险的
- 不可能执行穷尽测试
    + 覆盖完
    + 2个数的加法最大5位
        + 超过极限的显示情况
        + 数的类型--正，负，小数
    + 按照类型，风险级别
- 测试应该尽早启动，尽早介入
    + 瀑布模型
- 缺陷的群集现象
    + 业务越复杂，bug就越多
    + 如果需求不正确，后续的所有相关的都会出现bug
    + 和开发人员的技能相关----一般任务分配是按照模块划分的
    + 我们功能测试一般主要测试的流程和界面，界面的bug永远多于流程
- 杀虫剂悖论
    + 个人习惯问题，越熟悉，越容易忽略错误
    + 换种思想，调整思维，及时评审，交叉测试
- 测试活动依赖测试背景
    + 环境的不同
    + 着重点不一样，关注点不一样
- 不存在缺陷的谬论
