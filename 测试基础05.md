- 测试级别
    + 需求测试----需求评审
        + 完整性
        + 正确性
        + 一致性
        + 可行性
        + 无二义性
        + 健壮性
        + 必要性
        + 可测试性
        + 可修改性----
    + 单元测试
        + 组件，函数，类，数据逻辑，模型等各种代码逻辑
        + 对应的是详细设计文档--lld
        + 可以发现内存泄漏，算法冗余，分支覆盖率低，等各种效率低的问题
        + 需要测试人员有代码基础
        + 属于白盒测试
    + 集成测试---接口测试
        + 模块之间的数据交互
            + 内部接口和第三方接口
        + 对应文档是概要设计说明书-hld
        + 可以发现一些功能上无法发现的bug---转账转负数
        + 属于灰盒测试
    + 系统测试
        + 就是对整个系统的全面测试，包括很多项，大类包括，功能，兼容，性能，安全，用户体验等等方面的测试
        + 对应需求文档--SRS
        + 快，方便操作，主观上已于用户接受
        + 一般在市场上，通常将系统测试理解为功能测试？？
        + 属于黑盒测试
        + 注意：
            + 配置项测试：
                + 有软件提供方组织，由独立于软件开发人员实施
                + 系统测试由软件需求方组织，由独立于软件开发人员实施
                + 二组都可以委托第三方进行测评
    + 验收测试
        + α
            + 开发环境，
            + 内测
            + 通常开发在场
        + β
            + 用户环境
            + 公测
            + 通常开发不在场
        + λ
        + UAT
            + 测试环境
## 测试类型
- 功能测试
    + 检查错误，遗漏，冗余
    + 隐藏需求
    + 对输入输出进行验证是否正常响应----驱动
    + 一般使用手工测试；在开发提测后，首先使用功能验证
    + 功能测试在所有测试类型之前进行
- 性能测试
    + 主要因素---cpu，内存，读写
    + 验证在多用户同时使用下的系统的运行情况
        + 2 5 10
    + 性能测试属于自动化
    + 一般借助工具----LR，jmeter
    + 一天完成10W笔业务，单笔业务的时间；二八原则
